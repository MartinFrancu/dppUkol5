20 ## Build instructions
21 
22 **Inicpp** is build as shared and static library using `cmake`. There are two options to control which libraries are built, by default both are set to ON. Basic syntax is:
23 
24 ```
25 cmake [-G generator] [-DBUILD_STATIC=ON|OFF] [-DBUILD_SHARED=ON|OFF] source_dir
26 ```
27 
28 Also C++ compiler with at least C++14 support is required.
29 
30 ### Linux
31 
32 To build the library run following commands from root of source code directory:
33 
34 ```{.sh}
35 $ mkdir -p build
36 $ cd build
37 $ cmake ..
38 $ make
39 ```
40 
41 To build and run unit tests using Google Test framework run additional commands:
42 
43 ```{.sh}
44 $ make -f tests/Makefile
45 $ ./tests/run_tests
46 ```
47 
48 **Note:** For building tests you must have all the sources including git submodules. This could be done using `git clone --recursive https://github.com/SemaiCZE/inicpp.git` command when clonning or `git submodule update --init` when you already have the sources.
49 
50 ### Windows
51 
52 For Windows there are two ways of building `inicpp`. For both ways `cmake` has to be installed on machine.
53 
54 Using **MS Visual Studio**:
55 - As stated `Visual Studio 2015` (or later) should be installed on the machine.
56 - If dependencies are successfully fulfilled then run `cmake` in root directory of repository using:
57 ```
58 > cmake -G "Visual Studio 14 2015"
59 ```
60 - This command will generate solution files
61 - Open solution file `inicpp.sln` using `Visual Studio`
62 - Build `ALL_BUILD` project which will hopefully successfully build whole library
63 - Distribute static or shared binaries which can be found in target build directories to your program/library
64 
65 Using **MS Visual C++**:
66 - Besides `Visual C++ 2015` (or later) `nmake` compilation tool is needed (both should be part of `Windows SDK`)
67 - Run `cmake` in root directory of repository using:
68 ```
69 > cmake -G "NMake Makefiles"
70 ```
71 - This will generate file `Makefile` which can be interpreted with `nmake`
72 - Now enter following easy command:
73 ```
74 > nmake
75 ```
76 - After this compiled library should appear in working directory
77 - Depending on your needs use `inicpp` or `inicpp_static` in your library or program

97 ### Basic usage without schema
98 
99 Slightly modified version of basic example.
100 
101 ```{.cpp}
102 std::cout << "Load and parse config from string" << std::endl;
103 config example_conf = parser::load(get_config());
104 
105 std::cout << "Iterate through whole config and print it" << std::endl;
106 for(auto &sect : example_conf) {
107  std::cout << " Section: '" << sect.get_name() << "'" << std::endl;
108  // Iterate through options in a section
109  for(auto &opt : sect) {
110  std::cout << " Option: '" << opt.get_name() << "' with value(s): ";
111  if (!opt.is_list()) {
112  std::cout << "'" << opt.get<string_ini_t>() << "'";
113  } else {
114  for (auto &list_item : opt.get_list<string_ini_t>())
115  std::cout << "'" << list_item << "' ";
116  }
117  }
118 }
119 
120 std::cout << "Get number as signed_ini_t type" << std::endl;
121 try {
122  signed_ini_t number = example_conf["Numbers"]["num"].get<signed_ini_t>();
123  std::cout << " " << number << std::endl;
124 } catch (bad_cast_exception) {
125  std::cout << " Item 'num' in 'Numbers' section cannot be casted to signed_ini_t type" << std::endl;
126 }
127 
128 std::cout << "Change some values - could be properly typed" << std::endl;
129 // Make reference to the section
130 section &number_sect = example_conf["Numbers"];
131 // Change some values
132 number_sect["num"].set<signed_ini_t>(42222);
133 signed_ini_t new_num = number_sect["num"].get<signed_ini_t>();
134 std::cout << " Option 'num' in 'Numbers' section is '" << new_num << "'" << std::endl;
135 // Following two lines are equivalent
136 number_sect["num_oct"].set<string_ini_t>("0756");
137 number_sect["num_oct"] = "0756"s;
138 std::cout << " set method and assingment operator on option are equivalent" << std::endl;
139 
140 std::cout << "Change single value to list and vice versa" << std::endl;
141 option &num_opt = number_sect["num"];
142 num_opt.add_to_list<signed_ini_t>(99);
143 if (num_opt.is_list()) {
144  std::cout << " 'num' option in 'Numbers' section is list" << std::endl;
145 } else {
146  std::cout << " 'num' option in 'Numbers' section is single value" << std::endl;
147 }
148 // Remove item from the list (specifying position)
149 num_opt.remove_from_list_pos(0); // remove first item
150 std::cout << " first item from 'num' option list removed" << std::endl;
151 if (num_opt.is_list()) {
152  std::cout << " 'num' option in 'Numbers' section is list" << std::endl;
153 } else {
154  std::cout << " 'num' option in 'Numbers' section is single value" << std::endl;
155 }
156 std::cout << " 'num' option value is '" << num_opt.get<signed_ini_t>() << "'" << std::endl;
157 
158 std::cout << "Save changes to ostream and print the result" << std::endl;
159 std::ostringstream ofs;
160 ofs << example_conf;
161 std::cout << ofs.str();
162 ```
163 
164 Without schema all items are treated as string type, but runtime conversion could be done to one of supported types if possible.

166 ### Schema validation support
167 
168 Schema validation can bring you ability to make sure, that a config file contain what you expect with type safety. In addition, this implies better performance on heavy reading of validated configuration. Slightly modified version of schema validation example.
169 
170 ```{.cpp}
171 std::cout << "Create config schema" << std::endl;
172 schema schm;
173 
174 // add section 'Section 1'
175 section_schema_params section_1_params;
176 section_1_params.name = "Section 1";
177 section_1_params.comment = "comment";
178 section_1_params.requirement = item_requirement::optional;
179 schm.add_section(section_1_params);
180 
181 // add options to 'Section 1'
182 option_schema_params<signed_ini_t> option_1_params;
183 option_1_params.name = "Option 1";
184 option_1_params.type = option_item::single;
185 option_1_params.requirement = item_requirement::mandatory;
186 option_1_params.validator = [](signed_ini_t i){ return i < 0; }; // valid values are only negative ones
187 option_1_params.comment = "Important option\nshould be negative";
188 option_1_params.default_value = "-1";
189 schm.add_option("Section 1", option_1_params);
190 
191 option_schema_params<float_ini_t> option_4_params;
192 option_4_params.name = "float1";
193 option_4_params.type = option_item::single;
194 option_4_params.requirement = item_requirement::mandatory;
195 schm.add_option("Section 1", option_4_params);
196 
197 
198 std::cout << "Load and validate config in relaxed mode" << std::endl;
199 config conf = parser::load(get_config(), schm, schema_mode::relaxed);
200 
201 std::cout << "Check, if options are properly typed" << std::endl;
202 std::cout << " 'Option 1' is signed_ini_t type with value '" <<
203  conf["Section 1"]["Option 1"].get<signed_ini_t>() << "'" << std::endl;
204 std::cout << " 'float1' option has value '" << conf["Section 1"]["float1"].get<float_ini_t>() <<
205  std::endl;
206 std::cout << " 'unknown_option' was left as string with value '" <<
207  conf["Section 1"]["unknown_option"].get<string_ini_t>() << std::endl;
208 
209 std::cout << "Validation with strict mode fails due to 'unknown_option'" << std::endl;
210 try {
211  parser::load(get_config(), schm, schema_mode::strict);
212 } catch (validation_exception) {
213  std::cout << " Strict mode validation failed" << std::endl;
214 }
215 
216 std::cout << "Write default configuration from schema to stream" << std::endl;
217 std::ostringstream str;
218 str << schm;
219 std::cout << str.str();
220 
221 std::cout << "Write current configuration with comments from schema to stream" << std::endl;
222 str.str("");
223 parser::save(conf, schm, str);
224 std::cout << str.str();
225 ```
